## 二、算法集成接口规范（给算法工程师）

Demo 中 `js/app.js` 里的 `startCapture()` 函数标记了 **🔌 算法集成点**。

建议你们的动作检测算法按以下接口输出：

```javascript
// 每帧回调，传给前端
{
  handAccuracy: 0-100,      // 手势准确度（手形、位置、动作轨迹、朝向）
  faceExpression: 0-100,    // 表情/口动（MediaPipe Face Mesh）
  poseDirection: 0-100,     // 体态/方向
  overall: 0-100            // 综合评分
}
```

**集成方式优先级：**
1. **WebAssembly**（推荐）：将 MediaPipe Hands + Face Mesh 打包为 .wasm，浏览器直接运行
2. **WebSocket API**：算法部署为后端服务，前端通过 WebSocket 实时传输视频帧
3. **MediaPipe JS API**：直接使用 MediaPipe 浏览器端 JS 库

## 五、技术栈说明

| 层 | 当前 Demo | 正式开发建议 |
|----|----------|-------------|
| 前端框架 | 原生 HTML/CSS/JS | React / Vue + Three.js |
| 手语视频 | 占位符 | CDN 分发真人视频（H.265） |
| 动作检测 | 接口预留 | MediaPipe WASM / 自研算法 |
| 后端 | 无（纯前端） | Node.js / Spring Boot + MySQL + Redis |
| 3D 空间站 | 占位 CSS | Three.js / Unity WebGL |
| 部署 | 静态文件 | Vercel / 阿里云 OSS + CDN |
