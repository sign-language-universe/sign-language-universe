## 三、向特殊教育专家说明现状的话术

### 3.1 沟通原则

- **先给成果**：不要上来就说"我们还在早期"，先展示 Demo
- **用设计文档做桥梁**：专家熟悉设计文档 v3，用文档中的概念来解释 Demo
- **明确专家能贡献什么**：不是来"听汇报"，而是来"审核数据 + 提供方向"
- **坦诚但不焦虑**：承认这是快速原型，不是在交作业

### 会议中的关键问题清单

**问算法工程师（14:20-14:35）：**
1. 当前算法原型能检测到哪些维度？（手型？位置？方向？表情？）
2. 浏览器端跑 MediaPipe 的延迟大概多少？（是否需要降帧率）
3. 接口输出能否在 6月12日前对接到 Demo？如果不能，阻塞在哪？
4. 视频素材有什么格式要求？（分辨率、帧率、背景要求）

**问特殊教育专家（14:35-14:55）：**
1. Demo 里词汇的打法描述是否足够详细？
2. 评分时"表情"和"口动"需要分开计分吗？权重怎么设？
3. 50个核心词汇的优先级——按什么标准排？（使用频率？学习难度？场景覆盖？）
4. 有没有现成的真人手语视频素材可以先用来测试？

**问产品需求方（14:55-15:10）：**
1. 设计文档 v3 里有哪些需求是我们 Sprint 1 必须覆盖的？
2. 视频素材拍摄的时间线和预算？
3. 有没有可用的竞品参考（比如 SignTown、Lingvano 的截图）？
